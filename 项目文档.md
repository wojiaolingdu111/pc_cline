# PC 端本地文字转语音桌面应用方案（基于 Qwen3 TTS）

## 1. 方案目标

本方案用于构建一个运行在 **PC 端本地** 的文字转语音桌面应用，采用如下技术路线：

- **Tauri 2**：桌面应用框架
- **Vue 3 前端**：负责 UI 交互
- **Rust**：负责 TTS 推理、声音克隆、文件管理（原生实现，无 Python 依赖）
- **Qwen3 TTS (qwen3-tts-rs)**：作为核心 TTS / Voice Cloning 引擎

目标能力包括：

- 本地输入文本并生成音频
- 支持多个内置音色
- 支持用户克隆自己的声音
- 支持离线运行
- 支持本地播放和导出音频文件
- 支持后续扩展批量生成、语速调节、多语言等功能

---

## 2. 核心需求拆解

本项目的核心需求包括：

### 1）文字转音频
输入一段文本，系统在本地生成语音音频文件。

### 2）内置多个音色
应用提供若干预置音色，例如：

- 女声 1
- 女声 2
- 男声 1
- 男声 2
- 中性旁白

### 3）支持声音克隆
用户上传或录制一段自己的声音样本，系统生成一个可复用的个人音色 profile。

### 4）本地离线运行
模型推理、音频生成、profile 管理都在本机完成，不依赖远程 API。


---

## 4. 为什么选择 Rust + Qwen3 TTS

相较于 Python + Coqui TTS 路线，Rust 原生实现 Qwen3 TTS 的优势：

- **无需 Python 运行时**：用户无需安装 Python 或配置虚拟环境
- **无需 sidecar 管理**：没有进程管理、端口分配、心跳检测等复杂度
- **更好的性能和资源占用**：Rust 原生推理，内存占用更低
- **更简单的打包**：单一可执行文件，无需打包 Python 解释器和依赖库
- **跨平台一致性好**：没有 Python 依赖的跨平台差异问题

因此推荐架构：

```text
前端 (Vue 3) -> Tauri Commands (Rust) -> Qwen3 TTS (qwen3-tts-rs)
```

## 7. 功能模块设计

### 1）文本转语音模块

输入：

- 文本
- 音色 ID
- 语速
- 语言
- 输出格式

输出：

- 音频文件路径
- 或生成任务结果

### 2）内置音色模块

预置 speaker profile：

- `female_01` — 温柔女声
- `female_02` — 明亮女声
- `male_01` — 沉稳男声
- `male_02` — 清晰男声
- `narrator_01` — 中性旁白

实现方式：Qwen3 TTS 模型原生支持多说话人。

### 3）声音克隆模块

输入：

- 用户上传或录制的一段参考音频
- profile 名称
- 语言信息（可选）

处理过程：

- 校验音频格式
- 提取说话人特征（x-vector）
- 生成 voice profile 并保存元信息

输出：

- `voiceProfileId`
- profile 元信息

### 4）音频播放与导出模块

支持：

- 本地试听
- 重新生成
- 导出 `.wav`

### 5）任务管理模块

维护任务状态：

- `pending`
- `processing`
- `success`
- `failed`

## 8. 数据流设计

### 普通 TTS 流程

```text
前端输入文本
   ↓
选择内置音色
   ↓
调用 Tauri command
   ↓
Rust 调用 Qwen3 TTS 模型生成音频
   ↓
返回音频文件路径
   ↓
前端播放 / 导出
```

### 声音克隆流程

```text
用户上传或录制参考音频
   ↓
前端调用 clone_voice
   ↓
Rust 复制参考音频到 profiles 目录
   ↓
保存 profile 元信息
   ↓
返回 profile ID
   ↓
后续生成时使用 x-vector 提取的说话人嵌入
```

## 9. 核心接口设计

### `generate_speech`

输入示例：

```json
{
  "text": "你好，欢迎使用本地语音合成应用。",
  "voiceId": "female_01",
  "speed": 1.0,
  "language": "zh",
  "outputFormat": "wav"
}
```

输出示例：

```json
{
  "taskId": "tts-001",
  "status": "success",
  "audioPath": "/path/to/output.wav"
}
```

### `clone_voice`

输入示例：

```json
{
  "name": "my_voice",
  "audioPath": "/path/to/reference.wav",
  "language": "zh"
}
```

输出示例：

```json
{
  "voiceProfileId": "voice-user-001",
  "status": "success"
}
```

### `list_voices`

返回：

- 内置音色列表
- 用户克隆音色列表

### `delete_voice_profile`

删除某个用户自定义音色。

## 10. Rust 层职责设计

### `main.rs`
- 注册 Tauri commands
- 初始化应用状态
- 启动桌面应用

### `commands.rs`
- 暴露给前端的 command
- 参数校验
- Qwen3 TTS 模型调用
- 声音克隆逻辑
- 生成结果返回

### `file_manager.rs`
- 管理输出目录
- 管理 voice profiles
- 管理模型路径

### `state.rs`
- TTS 引擎（`TtsEngine`）包装 Qwen3 TTS 模型，支持懒加载
- GPU/CPU 自动检测
- 保存全局运行状态

### `license.rs`
- 授权管理（可选）

---

## 11. 状态管理

### `tts store`
- 当前文本
- 当前任务状态
- 当前音频路径
- 当前生成结果

### `voices store`
- 内置音色列表
- 用户音色列表
- 当前选中音色

### `settings store`
- 输出目录
- 默认格式
- 默认语速
- 当前语言
- TTS 引擎状态
